Reactive Programming with Actors

Johannes Pelto-Piri

Reactive Manifesto

  • Applications should:
    • React to events
    • React to load
    • React to failures
    • React to users

A actor based system will help you with this

Actors

  • An actor is a process with a state, a actor will process messages that has been sent to its mailbox.
  • A actor can only communicate by sending messages
  • A actor can supervise and create new actors.

They also:

  • Actors can create other actors and form a natural hierarchy
  • Actors can supervise other actors and repair, restart or terminate them when they fail.
  • Provide you with location transparency, so your system can span multiple machines easily

Designing a actor based system

  • Much like any organization actors will naturally form a top down hierarchy
    • A actor should protect itself by delegating slow and dangerous operations to another actor
  • Actors should not block, if you need to do a blocking operation, delegate that operation to a child
